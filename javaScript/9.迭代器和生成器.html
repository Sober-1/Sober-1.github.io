# 迭代器

# 迭代器和可迭代对象

TIP

迭代器：在容器对象（container，例如链表或数组）上遍访的接口，设计人员无需关心容器对象的内存分配的实现细节。

从迭代器的定义我们可以看出来，迭代器是帮助我们对某个数据结构进行遍历的对象。

在 JavaScript 中，迭代器是一个对象，并且该对象需要满足迭代器协议（iterator protocol）。

MDN 对迭代协议的介绍open in new window

迭代器协议定义了产生一系列值（无论是有限还是无限个）的标准方式，而在 js 中这个标准就是一个特定的 next 方法。

next 方法有如下的要求：

一个无参数或者一个参数的函数，返回一个应当拥有以下两个属性的对象：

done（boolean）

如果迭代器可以产生序列中的下一个值，则为 false。（这等价于没有指定 done 这个属性。）

如果迭代器已将序列迭代完毕，则为 true。这种情况下，value 是可选的，如果它依然存在，即为迭代结束之后的默认返回值。

value

迭代器返回的任何 JavaScript 值。done 为 true 时可省略。

如果返回了一个非对象值（比如 false 或 undefined），则会抛出一个 TypeError 异常（"iterator.next() returned a non-object value"）。

实际上，迭代器就是类似一个这样的对象：

var arr = [1, 2, 3, 4, 5, 6];
let index = 0;
var iterator = {
  next: () => {
    if (index < arr.length) {
      return { value: arr[index++], done: false };
    } else {
      return { value: undefined, done: true };
    }
  },
};

console.log(iterator.next()); // {value: 1, done: false}
console.log(iterator.next()); // {value: 2, done: false}
console.log(iterator.next()); // {value: 3, done: false}
console.log(iterator.next()); // {value: 4, done: false}
console.log(iterator.next()); // {value: 5, done: false}
console.log(iterator.next()); // {value: 6, done: false}
console.log(iterator.next()); // {value: undefined, done: true}

通过上述迭代器的特征，我们可以封装一个迭代器函数，当我们传入一个数组时，会生成一个该数组的迭代器，调用next 方法会依次返回数组中的元素。

function createIterator(arr) {
  let index = 0;
  return {
    next: () => {
      if (index < arr.length) {
        return { done: false, value: arr[index++] };
      } else {
        return { done: true, value: undefined };
      }
    },
  };
}

const arr1 = [1, 2, 3, 4, 5];
const arr1Iterator = createIterator(arr1);
arr1Iterator.next();
arr1Iterator.next();
arr1Iterator.next();

const arr2 = ["a", "b", "c", "d", "e"];
const arr2Iterator = createIterator(arr2);
arr2Iterator.next();
arr2Iterator.next();
arr2Iterator.next();

通过上面的封装，我们每次传入一个新的数组，就可以生成一个该数组的迭代器，通过调用迭代器内部的 next 方法，可以得到数组中的元素。

上述代码是一个有限的迭代器，什么叫做有限的迭代器呢？就是通过我们不断的调用 next，最终 done 的值一定会变成true，而迭代器是有无限迭代器的概念的，什么是无限迭代器呢？其实就是无论调用多少次 next，done 的值永远是 false。例如：

function infinityIterator() {
  let index = 0
  return {
    next: () => {
      return { done: false, value: index++ }
    }
  }
}

const iterator = infinityIterator()
iterator.next() // 0
iterator.next() // 1
iterator.next() // 2
...             // ...

虽然我们进行了封装，但感觉还是少了少了点东西，如果我们想使用迭代器，我们还要先创建一个变量，然后再把这个变量传入我们封装好的函数中，才能使用迭代器。真的好麻烦...那么有没有更好的方法呢？有没有可能我们在创建变量的同时自动帮我们生成了这个变量的迭代器呢？

没错！

可迭代对象就可以帮助我们实现，什么是可迭代对象呢？

  • 它和迭代器是不同的概念

  • 当一个对象实现了iterable protocol协议时，它就是一个可迭代对象

  • 这个对象的要求是必须实现@@iterator 方法，在代码中我们使用Symbol.iterator访问该属性

  • Symbol.iterator方法必须返回一个迭代器

示例如下：

const iterableObj = {
  value: [1, 2, 3, 4, 5],
  [Symbol.iterator]: function () {
    let index = 0;
    return {
      next: () => {
        if (index < this.value.length) {
          return { done: false, value: this.value[index++] };
        } else {
          return { done: true, value: undefined };
        }
      },
    };
  },
};

console.log(iterableObj[Symbol.iterator]);

TIP

break关键词可以帮助我们提前终止迭代器，只需要在可迭代对象的[Symbol.iterator]方法中返回一个return方法，在该方法中我们就可以监听迭代器的终止。该方法要求我们必须返回一个对象。

const iterableObj = {
  value: [1, 2, 3, 4, 5],
  [Symbol.iterator]: function () {
    let index = 0;
    return {
      next: () => {
        if (index < this.value.length) {
          return { done: false, value: this.value[index++] };
        } else {
          return { done: true, value: undefined };
        }
      },
      return: () => {
        console.log("我监听到了迭代器的终止");
        return { done: true, value: undefined };
      },
    };
  },
};

for (const value of iterableObj) {
  if (value === 2) break;
}

// 打印出'我监听到了迭代器的终止'

# 生成器

# 生成器和生成器函数

TIP

  • 生成器是由生成器函数产生的。
  • 生成器是一种特殊的迭代器。
  • 生成器主要运用于函数，是 ES6 中新增的一种函数控制的方案，它能够使我们灵活的控制函数暂停执行以及继续执行。

那么什么是生成器函数呢？

TIP

  • 生成器函数的声明方式和普通函数差不多，但是需要在function关键词后加一个*号。
  • 生成器函数可以通过yield关键字来控制函数的执行流程。
  • 生成器函数调用时并不会执行函数，它的返回值是一个生成器（Generator）。
  • 生成器函数调用既然返回的是生成器，而生成器又是一种特殊的迭代器，因此我们可以通过next关键字来调用生成器函数
function* generatorFun() {
  console.log(1);
  yield;
  console.log(2);
  yield;
  console.log(3);
  console.log(4);
  console.log(5);
}

const fun = generatorFun();
console.log(fun); // generatorFun {<suspended>}
fun.next(); // 1
fun.next(); // 2
fun.next(); // 3 4 5

# 生成器和迭代器的替代

既然生成器函数返回值是一个特殊的迭代器，那么我们上面自己封装的创造生成器的函数是否可以直接用生成器函数替代呢？

function* create(arr) {
  for (const item of arr) {
    yield item;
  }
}

const test = create([1, 2, 3, 4, 5]);
console.log(test.next()); // {value: 1, done: false}
console.log(test.next()); // {value: 2, done: false}
console.log(test.next()); // {value: 3, done: false}

后来我发现，还有一个关键字yield*，它是什么呢？

TIP

  • 该表达式用于委托给另一个 generator 或可迭代对象。
  • 该表达式迭代操作对象，并产生它返回的每个值。
  • 该表达式本身的值是当迭代器关闭时返回的值（即 done 为 true 时）

大概是什么意思呢？意思就是yield*表达式之后可以跟一个可迭代对象或者生成器函数，然后它就会依次迭代这个可迭代对象或者生成器函数，用代码来解释就是这样的：

function* g1() {
  yield 2;
  yield 3;
  yield 4;
}

function* g2() {
  yield 1;
  yield* g1();
  yield 5;
}

// 此时的g2相当于什么呢？它就相当于：
function* g2() {
  yield 1;
  yield 2;
  yield 3;
  yield 4;
  yield 5;
}
