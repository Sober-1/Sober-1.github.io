# 闭包的定义

闭包在计算机科学中和在JavaScript中是不同的

  1. 在计算机科学中闭包的定义

TIP

  • 闭包（Closure），又称词法闭包（Lexical Closure）或函数闭包（function Closure）
  • 是在支持头等函数的编程语言中，实现词法绑定的一种技术
  • 闭包在实现上是一个结构体，它存储了一个函数和一个关联的环境（相当于一个符号查找表）
  • 闭包跟函数最大的区别在于，当捕捉闭包时，它的自由变量会在捕捉时被确定，这样即使脱离了捕捉的上下文，它也能照样运行
  1. 在 MDN 中闭包的定义

TIP

  • 一个函数和对其周围状态（lexical environment，词法环境）的引用捆绑到一起（或者说函数被引用包围），这样的组合就是闭包
  • 也就是说，闭包可以让你在一个内层函数中访问到其外层函数的作用域
  • 在 JavaScript 中，每当创建一个函数，闭包就会在函数创建的同时被创建出来

是不是感觉似懂非懂，很抽象？

根据 mdn 的理解，我们可以认为：

如果一个普通的函数，可以访问外层作用域的自由变量，那么它就可以被称为闭包。

所以说从广义的角度来说，在 JavaScript 中任何一个函数都包含了闭包。

但是将计算机科学对闭包的定义结合起来我们可以发现，狭义的角度来说，闭包不只能访问外层函数作用域，而且它的自由变量在捕捉时会被确定，即使脱离了上下文，也能照常运行。我们根据一段代码来理解一下。

function out() {
  var name = "out";
  function inner() {
    console.log(name);
  }
  return inner;
}

var myFun = out();
myFun();

这段代码最终的结果是什么呢？ 会打印出 out 函数内部的变量name，在某些语言中，一旦 out 函数执行完毕，内部的变量便不可以访问，但是因为闭包的存在，inner 函数访问的 name 在解析时已经被确定是 out 函数的局部变量 name 了，即使 inner 函数是在外部执行，已经脱离了 out 函数的执行上下文，但是 out 函数内部的变量依然可以被访问到。

WARNING

所以说，很多人理解的闭包就是可以在外部访问到函数内部作用域的情况就叫闭包，这种情况只是狭义上的，准确的来说，每一个函数都生成了闭包。
