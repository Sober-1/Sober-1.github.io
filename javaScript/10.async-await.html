# async/await

# 历史遗留问题

假设现在有这么一个场景，我们需要调取接口 A拿到返回值a，

然后再利用a来作为接口 B 的参数来调取接口 B，拿到返回值b，

再利用b作为接口 C 的参数，来拿到返回值c，最后打印出结果，如果用之前的知识要怎么做呢？

先模拟三个接口：

function promise1() {
  return new Promise((resolve) => {
    resolve("a");
  });
}

function promise2(a) {
  return new Promise((resolve) => {
    resolve(a + "b");
  });
}

function promise3(b) {
  return new Promise((resolve) => {
    resolve(b + "c");
  });
}

当我们使用Promise时，虽然看起来没有那么的凌乱了，但是还是需要写很多行代码，看起来还是不够简洁，如果使用生成器函数会怎样呢？

生成器函数：

function* getResult() {
  const res = yield promise1();
  const res1 = yield promise2(res);
  const res2 = yield promise3(res1);
  console.log(res2);
}

/*
  getResult是一个生成器函数，调用它会生成一个生成器
*/
const generator = getResult();
/*
  调用生成器的next方法会依次执行内部代码，
  因为每个接口返回值都是一个Promise，因此调用next的返回值的value也是一个Promise对象。
  （类似 {value: Promise, done: false}）
  我们可以拿到这个对象调用.then方法
*/
generator.next().value.then((res) => {
  generator.next(res).value.then((res1) => {
    generator.next(res1).value.then((res2) => {
      console.log(res2); // abc
    });
  });
});

/* 将我们上边的代码封装一下 */
function getEndResult(generatorFun) {
  const generator = generatorFun();

  function exec(value) {
    const result = generator.next(value);
    if (result.done) {
      return result.value;
    }
    result.value.then((res) => {
      exec(res);
    });
  }
  exec();
}

getEndResult(getResult); // abc

# async 是什么

  • async 用于声明一个异步函数，是单词（asynchronous）的缩写

嗯？突然发现了不一样的东西，使用async关键字声明的函数，返回值会隐式的包裹一层Promise对象！

既然返回的是一个Promise对象，那我们肯定可以调用它返回值的then方法了，如果我们返回不同类型的值，它最终获取的结果是什么呢？

async function result() {
  // 1. 返回一个普通值
  return 1;
  // 2. 返回一个thenable对象
  return {
    then: () => {
      resolve(2);
    },
  };
  // 3. 返回一个Promise
  return new Promise((resolve) => {
    resolve(3);
  });
}

const promise = result();
promise.then((res) => {
  console.log(res); // 1. 1  2. 2  3. 3
});

我们可以得出结论，如果返回一个普通值，那么.then获取到的结果就是这个普通值，如果返回一个thenable 对象或者 Promise 对象，那么获取到的结果就是resolve()的结果。

总结：

TIP

async函数和普通函数的返回值不同

  • 如果返回一个普通值，会得到一个fulfilled状态的 Promise。.then 的结果是这个值本身。
  • 如果返回一个thenable 对象或者 Promise 对象，会得到一个pending状态的 Promise。.then 的结果是thenable 对象或 Promise 对象 resolve 的值。

# await 是什么

TIP

  1. await关键字只能在带有async关键字的函数内部使用，在外部使用时会报错。
  2. await的格式[返回值] = await 表达式
  3. await后面的表达式可以是一个Promise对象或任何要等待的值。
  4. await表示的是 等待 ，等待的是右侧的表达式结果。
  5. await后面如果跟的是一个Promise，并且该 Promise 的状态是fulfilled,则返回值是这个Promiseresolve 的值，如果该 Promise 的状态是rejected，则会抛出异常。如果是一个thenable对象，那么返回值是 thenable 对象 resolve 的值。如果是普通值，那么返回值是表达式本身的值。
  6. await会暂停当前异步函数的运行，直到右侧表达式执行完毕，在异步函数中，await 语句下面的代码会被放入异步队列中。
  7. await右侧表达式的的执行不受 await 影响。
  8. await在最新的ES标准中和Promise.resolve()语义一致。

# await 的返回值和执行顺序

// 1. await后跟Promise
async function rightPromise() {
  const res = await new Promise((resolve) => resolve("promise"));
  console.log(res);
}

rightPromise(); // promise

// 2. await后跟普通值
async function rightVariable() {
  const res = await 1;
  console.log(res);
}

rightVariable(); // 1

// 3. await后跟thenable对象
async function rightThenable() {
  const res = await { then: (resolve) => resolve("thenable") };
  console.log(res);
}
rightThenable(); // thenable

如果说await是因为等待右侧表达式的结果，才会导致 await 语句下面的代码暂停执行的话，如果右侧表达式是一个同步函数呢？又或者是一个普通值，还会影响下面代码执行吗？我们试一下！

我们发现，即使 await 后跟一个同步函数，它依旧会阻止后续代码的执行。这是因为await和Promise.resolve()语义一致。即上述代码等同于：

async function asyncFun() {
  Promise.resolve(syncFun()).then((res) => {
    console.log(res);
    console.log("我是异步函数里面的代码");
  });
}

从这里我们可以理解，await后跟的表达式，其实是作为resolve 的参数，传入到Promise.then()中，因此表达式不会被阻塞。

而执行完 await之后的内容，即 await 下面的代码其实相当于在Promise.then()的回调中执行，因此会被作为异步任务处理。

既然这样我们就可以这么来理解，await 后的表达式我们可以把它当做Promise 的回调函数的主体，不会加入到任务队列，会立即执行，而await 语句之后的代码，相当于Promise.then()回调的内容，会被加入到任务队列，异步执行。

这样就能解释清楚为什么 await 之后跟的是同步代码，却还是会异步执行。

WARNING

很多人说 async/await 是同步代码，其实它的本质还是异步，只不过它的写法看起来更像同步罢了。

# await 表达式的异常

如果 await 之后的表达式抛出了异常，会发生什么呢？

function err() {
  throw new Error("抛出异常");
}

async function errFun() {
  const res = await err();
  console.log(res);
}

errFun(); // Error: 抛出异常

很遗憾，当 await 之后的表达式抛出异常时，我们的异步函数会直接报错，如果我们想对错误的情况进行处理该怎么办呢？😨

这时候就要用到我们的try...catch语句了

function err() {
  throw new Error("抛出异常");
}

async function errFun() {
  try {
    const res = await err();
    console.log(res);
  } catch () {
    console.log('我可以对错误信息进行捕获')
  }
}

errFun(); // 我可以对错误信息进行捕获
