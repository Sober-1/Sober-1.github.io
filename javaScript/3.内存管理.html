# 内存管理

TIP

不管哪种编程语言，在代码执行过程中都需要分配内存，不同的是某些编程语言需要手动分配内存，有些编程语言自动分配内存

# 内存生命周期

  1. 分配要使用的内存空间（申请）
  2. 使用分配的内存空间(存放）
  3. 不需要使用时，进行内存释放（释放）

# 不同类型的编程语言

手动管理：比如C、C++以及早期的 OC，需要手动申请内存管理和释放（调用malloc和free函数）

自动管理：比如java、JavaScript、Python、Dart 等，可以帮助我们自动管理内存

# Js 中的内存管理

TIP

  • 在 JavaScript 中，如果是基本数据类型，会帮助我们在栈空间中分配内存
  • 如果是复杂数据类型，会帮助我们在堆空间分配内存，并将该变量的引用指向该块内存地址

# 垃圾回收

TIP

因为内存大小是有限的，所以当内存不需要的时候，我们应该对其进行释放，以腾出更多的内存空间，垃圾回收器（Garbage Collection）也称为GC

  1. 常见的 GC 算法 — 引用计数

    TIP

    当一个对象被引用时，内部会有一个记录引用次数的值， 比如此时 obj 被引用一次，那么它的引用计数就是 1， 当我们让 use.fatherName 不再引用 obj 时，obj 的引用计数会减少， 直至变成 0，它就会被垃圾回收

var obj = {
  name: "lee",
};
var use = {
  fatherName: obj,
};

弊端： 当两个对象相互引用时，如果忘记手动取消引用，那么这两个对象就会一直互相引用，不会释放，造成内存泄漏

  1. 常见的 GC 算法 - 标记清除

    TIP

    设置一个根对象（Root Object），垃圾回收器会定期从根对象开始查找，如果某些对象没有被引用到，就会把这些对象进行垃圾回收，该算法可以很好的解决循环引用问题

    js 引擎广泛采用标记清除法，V8 引擎为了更好的优化，还会配合一些其他算法一起使用
