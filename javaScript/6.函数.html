格式为function 函数名 (形参) { 函数体 }

function test() {
  console.log("test");
}

  1. 立即执行函数：格式通常为(function(形参){函数体})(参数)或(function(形参){函数体}(参数))，想要立即执行函数做到立即执行，要做到两点：
  • 该函数必须是函数表达式而不能是函数声明。
  • 函数后必须跟()

为什么一定要是函数表达式呢？如果是函数声明，js 引擎在解析函数时，发现末尾有()会认为格式错误，所以立即执行函数必须是函数表达式，通过下面这些方法，可以让 js 引擎把我们的函数视为函数表达式。

(function (num) {
  console.log(num); // 使用()运算符 123
})(123);

(function (num) {
  console.log(num); // 使用()运算符 123
})(123);

!(function (num) {
  console.log(num); // 使用!运算符 123
})(123);
+(function (num) {
  console.log(num); // 使用+运算符 123
})(123);
-(function (num) {
  console.log(num); // 使用-运算符 123
})(123);
~(function (num) {
  console.log(num); // 使用*运算符 123
})(123);

var a = (function (num) {
  console.log(num); // 使用=运算符 123
})(123);

# 函数中 this 的指向问题

TIP

  • 在常见的编程语言中，几乎都有this这个关键字（Objective-C 中使用的是 self），但是 JavaScript 中的 this 和常见的面向对象语言中的 this 不太一样： 常见面向对象的编程语言中，比如 Java、C++、Swift、Dart 等等一系列语言中，this 通常只会出现在类的方法中。也就是你需要有一个类，类中的方法（特别是实例方法）中，this 代表的是当前调用对象。
  • 但是 JavaScript 中的 this 更加灵活，无论是它出现的位置还是它代表的含义。

this 指向什么

我们首先可以先分成两大部分，分别时全局中 this 指向什么以及局部函数中 this 指向什么，我们先来看全局：

var name = "lee";
console.log(name); // lee
console.log(window.name); // lee
console.log(this.name); // lee

通过对比我们发现，在全局作用域中，通过浏览器测试我们发现，this 指向的就是 window 对象。

但是，开发中很少直接在全局作用于下去使用 this，通常都是在函数中使用。

而我们知道，在函数被调用时都会创建一个执行上下文，这个上下文记录着函数的调用栈、AO 对象等，this也是其中的一条记录。

我们利用三种不同的调用方式，调用同一个函数，来看一下效果：

// 方式1: 全局调用
function getThis() {
  console.log(this);
}

getThis(); // window

// 方式2: 作为对象的属性调用
var obj = {
  getThis,
};

obj.getThis(); // {getThis: f}

// 方式3: 通过call/apply调用
getThis.call(123); // Number {123}

我们发现，不同的调用方式，this 的指向也是不同的，这说明了什么呢？

  • 函数在全局调用时，JavaScript 会默认给 this 绑定一个值
  • 函数的 this 是在被调用时确定的。
  • this 的绑定和定义的位置（编写的位置）没有关系。
  • this 的绑定和调用方式以及调用的位置有关系

那么函数的 this 到底有几种绑定方式呢？

从上面的例子可以看出，当函数被某个对象进行调用时，该函数的 this 会指向调用它的对象，这种情况就是隐式绑定。

隐式绑定有一个前提条件：

  • 必须在调用的对象内部有一个对函数的引用（比如一个属性）；
  • 如果没有这样的引用，在进行调用时，会报找不到该函数的错误；
  • 正是通过这个引用，间接的将 this 绑定到了这个对象上；

如果我们不希望在某个对象的内部有对这个函数的引用，但是又想让函数的 this 指向某个对象，这时候就需要我们用到显示绑定。

# 显式绑定

显示绑定有三种方法，分别是call、apply、bind，我们先来看一下三者的异同。

  • 在 js 中，所有函数都可以使用call、apply、bind方法。
  • call、apply 是在原有函数上绑定 this，bind 会返回一个新的函数
  • call 的传参方式为参数列表，apply 的传参方式为参数组成的数组，bind 的第一个参数会作为绑定的 this，其余参数会作为参数。
var obj = {
  name: "obj",
  a: 10,
  b: 20,
};

function add(c) {
  console.log(this.a + this.b + c);
}

add(); // NaN
add.call(obj, 30); // 60
add.apply(obj, [40]); // 70
var newFun = add.bind(obj, 50);
newFun(); // 80

# new 关键字绑定

我们先来看一下，new的过程会发生什么：

1.创建一个全新的空对象

2.新对象的__proto__ = 构造函数的 prototype

3.将步骤 1 新创建的对象作为 this 的上下文 （this 的绑定在这个步骤完成）

4.如果函数没有返回其他对象，表达式会返回这个新对象

所以说new的过程也是可以绑定 this 的。

function Person(name) {
  console.log(this);
  this.name = name;
}

var p = new Person("lee"); // {}
console.log(p); // { name: 'lee' }

# 特殊场景的 this 绑定方式

  1. 忽略显式绑定 如果在显示绑定中，我们传入一个null或者undefined。那么这个显示绑定会被忽略，使用默认规则：
function foo() {
  console.log(this);
}

foo.call(null); // window
foo.apply(undefined); // window

  1. 间接函数引用

创建一个函数的间接引用，这种情况使用默认绑定规则。

  • 赋值(obj1.foo = obj.foo)的结果是 foo 函数
  • foo 函数被直接调用，那么是默认绑定

  console.log(this);
}

var obj = {
  name: "obj",
  foo: foo,
};

  name: "obj1",
};

  1. 箭头函数

箭头函数不使用 this 的四种标准规则（也就是不绑定 this），而是根据外层作用域来决定 this。

var arrowFun = () => {
  console.log(this);
};

var obj = {
  arrowFun: arrowFun,
};

arrowFun(); // window
arrowFun.call("123"); // window
obj.arrowFun(); // window

# this 指向的优先级

既然我们知道了这么多绑定 this 的方法，那么它们的优先级谁更高呢？

1.默认规则的优先级最低

2.显示绑定优先级高于隐式绑定

3.new 绑定优先级高于隐式绑定

4.new 绑定优先级高于 bind

  • new 绑定和 call、apply 是不允许同时使用的，所以不存在谁的优先级更高
  • new 绑定可以和 bind 一起使用，new 绑定优先级更高

var name = "window";

  name: "person",

    console.log(this.name);
  },
};

function fun() {
  var f = person.fun;
  f(); // window
  person.fun(); // person
  (a = person.fun)(); // window

首先f()是独立函数调用，它的 this 指向 window，而person.fun()是隐式绑定，因此它的 this 指向 person，而(a = person.fun)()属于特殊的间接函数引用，它相当于独立函数调用，因此也会指向 window。

var name = "window";

var person1 = {
  name: "person1",
  foo1: function () {
    console.log(this.name);
  },
  foo2: () => console.log(this.name),
  foo3: function () {
    return function () {
      console.log(this.name);
    };
  },
  foo4: function () {
    return () => {
      console.log(this.name);
    };
  },
};

var person2 = { name: "person2" };

person1.foo1(); // person1
person1.foo1.call(person2); // person2

person1.foo2(); // window
person1.foo2.call(person2); // window

person1.foo3()(); // window
person1.foo3.call(person2)(); // window
person1.foo3().call(person2); // person2

person1.foo4()(); // person1
person1.foo4.call(person2)(); // person2
person1.foo4().call(person2); // person1

  1. 第一个，这是很明显的隐式绑定，因此 this 指向 person1

  2. 第二个，通过显式绑定，将 foo1 的 this 绑定到了 person2，因此打印出 person2

  3. 第三个，foo2 是箭头函数，不受隐式绑定的约束，this 指向上级作用域的 this，因此指向 window

  4. 第四个，箭头函数也不受显式绑定的约束，同上

  5. 第五个，foo3 执行返回了一个普通函数，执行这个普通函数相当于独立函数调用，因此指向 window

  6. 第六个，foo3 的 this 通过显式绑定，绑定到了 person2 上，但是 foo3 返回出来的函数还是相当于独立调用，同上

  7. 第七个，把 foo3 返回的函数通过显式绑定，this 指向了 person2，因此打印出 person2

  8. 第八个，foo4 返回了一个箭头函数，执行之后，此时的 this 指向箭头函数上级作用域的 this，也就是 foo4 的 this，而 foo4 的 this 指向 person1，因此打印出 person1

  9. 第九个，foo4 通过显式绑定，this 变为了 person2，那么这时候 foo4 返回的箭头函数的 this 也应该指向 foo4 的 this，也就是 person2

  10. 第十个，foo4 先执行，返回了箭头函数，然后箭头函数通过显式绑定 this，绑定 person2，但是箭头函数不受此影响，还是继续指向 foo4 的 this，也就是 person1
