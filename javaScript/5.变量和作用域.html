# 变量和作用域

TIP

在 JavaScript 中, 作用域为可访问变量，对象，函数的集合。

作用域的字面意思就是，代码块只在某一区域起作用。

# 了解 var、let、const

在学习作用域之前，我们首先要了解 js 中常用声明一个变量或常量的方式。

在 ES6 之前，我们的代码中，声明一个变量的方式，无论是字符串或者是对象等。都是使用 var 变量名 = xxx 的格式。直到 let 和 const 的出现，我们的代码才更加规范起来。

那么它们直接有什么区别呢？

var：声明一个变量，在该变量被声明之后，可以在下文代码中对该变量进行重新赋值，并覆盖原有值。

// 输入
var a = 123;

a = 456;

console.log(a); // 456

let：声明一个变量，和var的作用基本一致，不过使用let声明的变量，只在当前块级作用域可以使用。

const: 和let基本一致，都是只在块级作用域生效，但表示声明一个常量，顾名思义，常量代表着不可以被改变的值，一旦被赋值，就不可修改,否则就会抛出异常。

WARNING

这也就意味着，通过let，const声明的变量或者常量，不会存放在window对象以及全局的this中。

// 输入
const BASE = 1;

BASE = 2;

console.log(BASE); // Uncaught TypeError: Assignment to constant variable.报错

# 作用域和作用域链

  • 在了解全局变量以及局部变量之前，我们需要先了解什么是全局作用域以及局部作用域，以及什么叫做作用域链。

TIP

全局作用域

  • 一个js 文件或一个script 标签之中的内容，都可以称之为全局，这一块区域之中定义的所有变量或常量，都处于全局作用域中，这个文件共享某个变量或常量。
  • 全局作用域在页面打开的时候创建，在页面关闭时销毁，全局作用域其实就是全局执行上下文中的 GO。
  • 在全局作用域中有一个全局对象 window（代表的是一个浏览器的窗口，由浏览器创建），可以直接使用。如我们经常使用的console以及alert函数，就是 window 对象中的函数。

WARNING

  • 有一种特殊情况，在 ES5 以及之前，所有没有通过var 关键字声明的变量都会默认进行变量提升，成为全局变量。

TIP

局部作用域

  • 局部作用域，顾名思义，就是代表只作用在某一块区域内，在局部作用域定义的内容，只在这一区域可以访问，外部无法访问局部作用域。
  • 局部作用域也可以称为函数作用域，因为局部作用域是在函数调用的时候生成，在函数调用完成后销毁，而函数作用域其实就是函数的 AO 对象以及 this(可以绑定)。
  • 每调用一次函数就会创建一个新的函数作用域，它们之间是相互独立的。

TIP

作用域链

  • 当我们查找某个变量或属性时，查找的路径都是按照作用域链来查找，直至找到 GO 为止
  • 作用域链其实就是自身上下文VO加上父级的作用域。
  • 了解了全局作用域以及局部作用域，我们就可以大概了解什么是全局变量以及局部变量了，在全局作用域中声明的变量就是全局变量，在局部作用域声明的变量就是局部变量

# 全局变量示例

var foo = 123;

bar = 456;

function checkGlobalObject() {
  console.log(foo);
  console.log(bar);
}

checkGlobalObject(); // 123  456

# 块级作用域

TIP

其实除了局部作用域和全局作用域，在 ES6 中有了一个新的概念——块级作用域，具体是什么意思呢？顾名思义，块级作用域的意思就是每个代码块都有自己的作用域，比如一个大括号包裹起来的内容就是一个代码块，它有什么用呢？

我们先来看这段代码

var a = 1;

function test() {
  console.log(a);
  if (true) {
    var a = 123;
    console.log(a);
  }
}

test(); //  ?

上述代码会打印出什么内容呢？很多人都会觉得是 1 和 123？

其实并不是，上述代码会打印出undefined 和 123。

为什么会打印出undefined呢😮？

原因就是，当我们执行 test 函数时，因为是在局部作用域中访问变量 a，因此会先去局部作用域中寻找变量 a，那局部作用域中有没有变量 a 呢？我们可以看到 if 语句中声明了变量 a，但是当我们访问变量 a 时，还没有执行到这一步，但是有变量提升这个东西啊，此时在局部作用域中变量 a 已经进行了变量提升了，当我们在局部作用域中访问变量 a 时，局部作用域是这样的var a = undefined，因此第一次打印会打印出 undefined，而第二次打印时因为已经执行完毕了var a = 123，所以会直接打印出 123。

我们先来看看上述代码中如果我们使用let或者const会怎样呢？

var a = 1;

function test() {
  console.log(a);
  if (true) {
    let a = 123;
    // const a = 123
    console.log(a);
  }
}

test(); //  1   123

看到这很多人会问，就换个单词差距这么大？为什么第一次直接用了全局变量 a 呢？

首先我们需要了解一个知识点——暂时性死区

TIP

  • 暂时性死区

The variables are created when their containing Lexical Environment is instantiated but may not be accessed inany way until the variable’s LexicalBinding is evaluated.

译文：

当程序的控制流程在新的作用域（module function 或 block 作用域）进行实例化时，在此作用域中用 let/const 声明的变量会先在作用域中被创建出来，但因此时还未进行词法绑定，所以是不能被访问的，如果访问就会抛出错误。因此，在这运行流程进入作用域创建变量，到变量可以被访问之间的这一段时间，就称之为暂时死区。

这说明啥呢？说明其实在内部我们已经创建了局部作用域的变量 a，只是此时还不能被访问，因此在我们第一次打印变量 a 时，访问不到局部变量 a，只能去上级逐级寻找，在全局变量中找到了变量 a，因此第一次打印出来的就是全局变量 a(1)，而第二次就会找自己的局部变量了，就会正常打印出 123

因为暂时性死区的原因，使得我们通过let和const声明的变量或常量只能在同级的代码块中使用，这也就形成了所谓的块级作用域。我们可以简单理解为括号包裹的内容就形成一个块级作用域，比如if 语句,for 循环等。

WARNING

很多人都说let和const不会进行变量提升，这种说法其实是错误的，只是因为暂时性死区的存在，导致我们感官上不是特别明显，但是说它们不会进行变量提升是错误的。

我们再来看一个例子：

var a = "global";

function global() {
  console.log(a);
}

function test() {
  var a = "local";
  global();
}

test(); // 全局变量

这段代码的 global 函数会打印出来什么呢？

肯定很多人觉得会打印出local，因为 global 函数在 test 函数内执行，而 test 函数的 AO/VO 中有一个变量 a 为 local。

但是它打印出来的是global，为什么呢？在作用域链的概念中我们说，查找一个变量会顺着作用域链查找，而作用域链其实就是自身上下文和父级作用域，我们只需要找出来 global 函数的作用域链就知道为什么了。

我们找函数的父级作用域时，不要看函数在哪里被执行，而要看函数在哪里被定义，我们可以看到函数 global 处于全局变量中，因此，函数 global 的父级肯定是全局，也就是 GO。

我们再看 global 函数自身的 VO 都有什么，我们可以发现，它的 AO/VO 是空的，所以当执行 global 函数打印变量 a 时，先去 VO 寻找，发现没有，顺着作用域链去 GO 中寻找，发现 GO 中变量 a 的值为'global'，所以打印出了 global。
